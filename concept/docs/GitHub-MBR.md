# MBR Q1 FY26 | ATU 13 GitHub Copilot Acceleration

## Executive Summary

ATU 13's GitHub Copilot acceleration strategy centers on **earning customer trust through targeted technical unblocking**, rather than broad product evangelism. We've achieved 63% GHCP penetration across the territory by deliberately pairing demonstrations with tools customers already use, delivering hands-on proof sessions that move from interest to adoption, and identifying platform friction (CI/CD, security configuration, EA billing models) that unlocks both GitHub platform migration and Copilot consumption growth.

## Cross-Territory Trends

1. **Leading with hands-on proof (HoK, POC, Innovation Factory)** — deep technical engagements that demonstrate value in customer-specific contexts are converting faster than awareness campaigns
2. **Pairing Copilot with customer toolchains** — showing Playwright, JFrog, and security tools integrated with Copilot drives immediate practical understanding
3. **Solving technical blockers first** — fixing CI/CD pipelines and security configurations creates trust that accelerates broader platform adoption
4. **Cross-solution orchestration** — joining data and app-focused customer calls surfaces new GitHub opportunities within existing accounts
5. **EA-to-metered conversion at renewal** — identifying anniversary dates and converting enterprise agreement customers to consumption-based billing

# What We're Doing to Accelerate GitHub Copilot and Measurable Impact

We're executing a **trust-through-unblocking motion** across ATU 13: identify specific technical friction, deliver targeted solutions, and expand from that foundation. This contrasts with traditional product-first pitches and has proven more effective at driving both initial adoption and expansion.

## 1. Lead with Hands-On Proof (HoK, POCs, Innovation Factory)

Rather than high-level product discussions, we're delivering **scenario-based technical proof** that customers can immediately apply. This includes Innovation Factory end-to-end POCs, CSU-led collaboration engagements, and targeted use case demonstrations.

**What's working (examples):**

- **TriNet**: Conducted deep technical POC with Director of Engineering Productivity on **Custom Instructions at enterprise/org/repo scope** and **Model Context Protocol (MCP) servers** for secure coding standards and AWS compliance integration. Recorded multiple PoV sessions with GitHub TCS and Microsoft SMEs throughout Oct-Nov 2025. POC progressing to production deployment.

- **FinThrive**: Delivered focused **GitHub Copilot Coding Agent (GA)** demonstrations to Chief Quality Officer and VP Platform Engineering, positioning Copilot as part of broader **Azure DevOps → GitHub migration**. Engagement led to migration planning discussions and Coding Agent evaluation (Nov 2025).

- **Insight Enterprise**: Partnered with CSU to deliver **turn-key Secret Scanning configuration for SecOps team** via VBD. Result: **MoM growth in both Secret Scanning and GitHub Copilot seat expansion** following unblock.

## 2. Targeted Buyer Engagement (TDM-Focused Conversations)

We're identifying and directly engaging customer TDMs with **specific Software and Agentic AI value discussions**, rather than broad awareness campaigns. Key questions: "Where do you see value in AI Coding Agent? How are you getting value out of GHCP? Where can we help upskill and demonstrate value within your org?"

**What's working (examples):**

- **TriNet** (John Morales, Director Engineering Productivity): Multi-session technical dialogue on enterprise-scale Copilot deployment, resulting in Custom Instructions + MCP server POC and ongoing enablement partnership.

- **FinThrive** (Phil Stork, CQO / Adam Robinson, VP Platform Engineering): Platform-strategy engagement tying Copilot Coding Agent to ADO migration, opening dual-motion opportunity.

- **McKinstry**: Post-meeting enablement follow-up with GitHub Universe 2025 feature announcements and use case mapping, moving customer from awareness to active onboarding (Nov 2025).

## 3. Platform Friction Removal (CI/CD, Security, EA Blockers)

Solving **technical blockers and configuration challenges first** builds trust that accelerates broader platform adoption. We're addressing CI/CD pipeline issues, GHAS configuration, and EA billing model friction.

**What's working (examples):**

- **Lighthouse Document**: Resolved CI/CD pipeline issues and **GitHub Advanced Security (GHAS) configuration problems**. Customer immediately enabled **GHAS across all organizations** following technical unblock, demonstrating trust-driven expansion.

- **Cotiviti**: Unblocked **EA quote issue for 100 new licenses** (procurement via SHI PO, Azure billing transition clarification). Result: customer **committed to full metered conversion at upcoming anniversary** (Oct-Nov 2025).

- **Cooley LLP**: Addressed formal compliance inquiry on **GitHub Copilot AI model usage, data residency, zero-data retention** aligned to Microsoft Responsible AI Standard. Cleared legal/compliance blocker for deployment (Dec 2025).

## 4. Tool-Paired Demonstrations (Playwright, JFrog, Security Tools)

Instead of high-level capability overviews, we're delivering **real-world demonstrations of GitHub Copilot paired with tools customers already use** (Playwright for testing, JFrog for artifact management). These specific scenarios help customers see practical value immediately.

**What's working:**

- Customers consistently request "show me how this works with [our tool]" rather than generic demos
- Playwright + Copilot testing demonstrations drive faster adoption understanding than abstract testing capability discussions
- JFrog + Copilot artifact and security workflow demos resonate with DevSecOps-focused customers

## 5. Cross-Solution Orchestration (Joining Data/App Calls)

Engaging in **cross-solution customer conversations** (data, app modernization, security) surfaces new GitHub opportunities within existing accounts. Mentioning GitHub Copilot engagements on parallel solution calls has led to secondary follow-ups and multi-solution bundling.

**What's working:**

- Joining data or app-specific customer calls and offering GitHub Copilot context leads to follow-up conversations either directly or with adjacent teams within the customer organization
- Multi-threaded account engagement creates broader platform adoption opportunity

## 6. Identifying GitHub Platform Migration Opportunities

We're **proactively identifying customers on Azure DevOps or other SCM platforms** and positioning GitHub migration as an enabler for Copilot and metered GitHub services. Working more closely with GitHub account teams to orchestrate migration + Copilot motions.

**What's working (examples):**

- **FinThrive**: ADO → GitHub migration conversations opened door to Copilot Coding Agent evaluation and broader platform modernization discussion

## 7. EA-to-Metered Conversion at Anniversary

We're **identifying customers with upcoming EA anniversaries** who currently purchase GitHub and GitHub Copilot through enterprise agreements, and working with GitHub team + deal desk to convert to metered consumption models.

**What's working (examples):**

- **Cotiviti**: After unblocking 100-seat EA license issue, customer expressed willingness to convert to **all-metered purchase at upcoming anniversary**
